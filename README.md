# 🐋 Walrus Notes

Веб-приложение для организации и управления заметками с расширенными возможностями компоновки и визуализации связей.

## ✨ Возможности

### 📝 Управление заметками

- **Создание** - быстрый старт новых заметок
- **Редактирование** - богатый текстовый редактор
- **Удаление** - с подтверждением действия
- **Поиск** - мгновенный поиск по содержимому

### 🗂️ Система лейаутов

- **Создание лейаутов** - группируйте заметки по проектам/темам
- **Гибкая организация** - переносите заметки между лейаутами
- **Визуальное представление** - просматривайте структуру лейаутов

### 🌐 Совместная работа

- **Общие заметки** - делитесь заметками с другими пользователями
- **Уровни доступа** - настройка прав для совместной работы

### 📊 Графы заметок

- **Визуальные связи** - просматривайте связи между заметками
- **Навигация по графам** - интерактивное исследование отношений
- **Автоматическое построение** - система сама определяет связи

### Установка и запуск

```bash
git clone https://github.com/lkrassi/walrus-notes.git
cd walrus_notes

pnpm i

npm install

npm run dev

npm run build

npm run preview
```

Приложение будет доступно по адресу http://localhost:5173

## 🧱 Архитектура (FSD migration)

Проект мигрируется на подход Feature-Sliced Design с целевой структурой слоёв:

- `app` — инициализация приложения, провайдеры, роутинг, store
- `pages` — страничные композиции
- `widgets` — крупные составные UI-блоки
- `features` — пользовательские сценарии
- `entities` — бизнес-сущности и их модель
- `shared` — переиспользуемые UI/утилиты/конфиги

### Правила зависимостей слоёв

Зависимости разрешены только сверху вниз:

- `app` → `pages | widgets | features | entities | shared`
- `pages` → `widgets | features | entities | shared`
- `widgets` → `features | entities | shared`
- `features` → `entities | shared`
- `entities` → `shared`
- `shared` → только `shared`

Контроль выполняется через ESLint правила:

- `import/no-restricted-paths`
- `import/no-cycle`
- `boundaries/element-types`
- `unicorn/filename-case`
- `react/jsx-pascal-case`

### Алиасы

Поддерживаются оба стиля импортов (для плавной миграции):

- новый: `@/app`, `@/pages`, `@/features`, `@/entities`, `@/shared`, `@/widgets`
- legacy: `app`, `pages`, `features`, `shared`, `widgets`

### Нейминг

- React-компоненты: `PascalCase`
- Хуки и утилиты: `camelCase`
- Публичный API слайса: `index.ts`

### Размещение кода и дедупликация

- `shared` — только универсальные и повторно используемые примитивы (UI, hooks, utils), без знания про домен и провайдеры верхних слоёв.
- `widgets` — композиция и экранные блоки, которые могут использовать `shared`, но не наоборот.
- Если компонент/хук используется в разных виджетах и фичах — переносим в `shared` и импортируем через его public API.
- Если компонент относится к конкретному экранному блоку (sidebar/header/fileTree) — оставляем в `widgets`.
- Для похожих компонентов сначала пытаемся обобщить через пропсы/композицию; если смысл разный, оставляем отдельные реализации.
- При любом переносе/удалении дублей обязательно прогоняем `pnpm lint` и `pnpm type-check`.
